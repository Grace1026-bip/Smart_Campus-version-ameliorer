SMART FACULTY
02.04 – Architecture des API REST
Introduction
Les API REST constituent le lien entre le frontend développé avec Flutter et le backend développé avec FastAPI. Elles permettent l’échange sécurisé des données entre les utilisateurs et le serveur.
Chaque fonctionnalité de Smart Faculty est accessible à travers une ou plusieurs routes API.
Objectifs des API
Les API REST ont pour objectifs :
assurer la communication entre Flutter et FastAPI ;
sécuriser les échanges de données ;
centraliser les traitements métier ;
faciliter l’évolution du système.

Format des échanges
Toutes les API communiquent au format JSON.
Chaque requête contient les informations nécessaires au traitement.
Chaque réponse renvoie :
un statut ;
un message ;
les données demandées si l’opération réussit.
Sécurité des API
Toutes les routes protégées nécessitent une authentification.
Le backend vérifie :
l’identité de l’utilisateur ;
son rôle ;
ses permissions.
Aucune opération sensible n’est exécutée sans autorisation.
Conclusion
Les API REST constituent le point de communication central entre le frontend Flutter et le backend FastAPI. Elles garantissent une architecture moderne, sécurisée et évolutive permettant à Smart Faculty de répondre efficacement aux besoins de ses utilisateurs.
