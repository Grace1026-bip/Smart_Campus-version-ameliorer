SMART FACULTY
02.02 – Conception de la base de données
Introduction
La base de données constitue le cœur de Smart Faculty. Elle assure le stockage, l’organisation et la gestion de toutes les informations académiques nécessaires au fonctionnement de la plateforme.
Cette conception a pour objectif de garantir la cohérence des données, d’éviter les redondances et de faciliter les échanges entre le frontend Flutter et le backend FastAPI.
Le modèle retenu est une base de données relationnelle implémentée avec MySQL.
Objectifs de la base de données
La base de données devra permettre de :
stocker les informations des utilisateurs ;
gérer les promotions académiques ;
gérer les cours ;
enregistrer les notes ;
calculer les résultats ;
gérer les publications académiques ;
gérer les réclamations ;
enregistrer les notifications ;
conserver les alertes académiques ;
assurer la cohérence des informations.
Principales entités
La base de données sera composée des principales entités suivantes :
Utilisateurs
Cette table contient les informations communes à tous les utilisateurs.
Exemples d’informations :
identifiant ;
nom ;
prénom ;
adresse e-mail ;
mot de passe ;
rôle ;
statut du compte.
Étudiants
Cette table contient les informations propres aux étudiants.
Elle est liée à la table Utilisateurs.
Elle contient notamment :
matricule ;
promotion ;
année académique.
Enseignants
Cette table contient les informations spécifiques aux enseignants.
Elle est liée à la table Utilisateurs.
Elle permet d’identifier les cours assurés par chaque enseignant.
Promotions
Cette table représente les différentes promotions de la faculté.
Exemples :
L1 Informatique
L2 Informatique
L3 Informatique
M1 Informatique
Chaque promotion regroupe plusieurs étudiants et plusieurs cours.
Cours
Cette table contient :
le nom du cours ;
le volume horaire ;
le nombre de crédits ;
la promotion concernée ;
l’enseignant responsable.
Évaluations
Chaque cours peut comporter plusieurs évaluations.
Par exemple :
Interrogation
TP
Examen
Projet
Chaque évaluation possède une pondération.
Notes
Cette table enregistre les notes obtenues par les étudiants pour chaque évaluation.
Elle est directement liée :
à l’étudiant ;
au cours ;
à l’évaluation.
Publications (Valve)
Cette table contient toutes les annonces publiées par les enseignants.
Elle permet de diffuser :
annonces ;
documents ;
informations importantes.
Chaque publication appartient à un cours.
Réclamations
Cette table contient toutes les réclamations créées par les étudiants.
Chaque réclamation est liée :
à un étudiant ;
à un cours ;
à un enseignant.
Notifications
Cette table enregistre toutes les notifications envoyées par le système.
Chaque notification est destinée à un utilisateur.
Alertes académiques
Cette table contient les étudiants détectés comme étant en difficulté.
Les alertes sont générées automatiquement par le système.
Relations entre les entités
Les principales relations sont les suivantes :
Une promotion possède plusieurs étudiants.
Une promotion possède plusieurs cours.
Un enseignant peut enseigner plusieurs cours.
Un cours peut contenir plusieurs évaluations.
Une évaluation possède plusieurs notes.
Un étudiant possède plusieurs notes.
Un cours possède plusieurs publications.
Un étudiant peut créer plusieurs réclamations.
Un utilisateur peut recevoir plusieurs notifications.
Contraintes d’intégrité
Afin de garantir la cohérence des données, les règles suivantes seront appliquées :
chaque utilisateur possède une adresse e-mail unique ;
chaque étudiant possède un matricule unique ;
un cours appartient obligatoirement à une promotion ;
une note ne peut exister sans étudiant, cours et évaluation ;
une réclamation est obligatoirement liée à un cours ;
une publication appartient obligatoirement à un cours.
Perspectives d’évolution
La structure de la base de données a été conçue afin de permettre l’ajout futur de nouveaux modules sans remettre en cause son organisation.
Par exemple :
gestion des stages ;
gestion des TFC ;
assistant académique intelligent ;
Campus Analytics ;
gestion des soutenances.
Conclusion
La conception de la base de données constitue une étape essentielle dans la réalisation de Smart Faculty. Elle garantit une organisation cohérente des informations académiques et assure la stabilité du système.
Cette structure servira de référence lors de l’implémentation de la base de données MySQL et du développement des différents modules de l’application.
